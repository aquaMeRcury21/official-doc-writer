## 模式一：起草新公文

用户给出白话需求，需要新写一份公文时执行此流程。

### 流程

1. **文种判断**：加载 `official-document-writing` 技能，执行文种匹配
2. **需求采集**：按四阶段工作流采集信息，同步识别所有结构化字段（时间、地点、人名、金额、数据等）
3. **知识库检索**：加载 `doc-rag` 检索相关范文
4. **逐节构建**：逐节确认正文，同步累积结构化字段清单
5. **三步输出**：
   - ① 写 `_data/公文数据.json`（结构化字段）
   - ② 写 `_data/XX模板.txt`（正文 prose + `{{占位符}}`）
   - ③ 写 `.txt` → `knowledge-base/archive/`
   - ④ 写 `.docx` → 年份目录
   - ⑤ 运行 `python utils/generate_doc.py --dir 事件文件夹 --doc all` 验证

### 数据文件规范

`_data/公文数据.json` 包含：
- `docType`: 文种
- `title`: 标题
- `fields`: 键值对字段（时间/地点/人名/金额等）
- `lists`: 列表数据（参会单位/议程/选手名单等）
- `meta`: 元数据（发文字号/发文机关等）
- `templates`: 文种→模板文件名映射

`_data/XX模板.txt` 包含正文 prose，结构化数据处用 `{{字段名}}` 占位。

## 模式二：修改已有公文

用户说改某份已有公文时执行此流程。

### 流程

1. **定位文件夹**：根据用户描述查找对应事件文件夹
2. **读取现有文件**：
   - 读取 `_data/公文数据.json`
   - 读取对应的 `_data/XX模板.txt`
3. **判断修改类型**：

   | 要改什么 | 改哪个文件 |
   |---------|-----------|
   | 人名/时间/地点/金额/数据 | 改 `公文数据.json` 对应字段 |
   | 措辞/语气/排比/段落结构 | 改 `XX模板.txt` 对应位置 |
   | 两者都有 | 分别改两个文件 |

4. **精确定位替换**：只改目标内容，不重写整篇
5. **用户确认**：展示修改效果
6. **重新生成**：
   ```
   python utils/generate_doc.py --dir 事件文件夹 --doc 文种
   ```
7. **更新归档**：同步更新 .txt 到 knowledge-base/archive/

### 特殊场景

- **无数据文件的老公文**：先逆向提取结构化数据，创建 `_data/公文数据.json` 和 `_data/XX模板.txt`，再走修改流程

---

## 通用约束

- 所有文种适用，不分格式是否固定
- 每次输出必带三样东西：数据 + 模板 + 成品
- 修改时优先改数据文件或模板，不直接改 .docx
- 数据文件为后续修改的唯一入口
