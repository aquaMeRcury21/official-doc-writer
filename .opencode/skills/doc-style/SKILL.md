# 公文写作规范

## 斩断 AI 腔调

### 严禁使用的机械连接词

- "首先……其次……最后……"
- "总而言之""综上所述""众所周知"
- "显而易见""毋庸置疑"
- "从某种程度上说""在一定意义上"

### 严禁使用带有强烈个人情感的形容词

- 如"令人振奋的""十分感人的""极其重要的"
- 如"突出的贡献""卓越的成就"（改为"显著成绩""积极进展"）

### 必须多用公文专有词

- 拟、悉、函告、届时、特此、责成、业经、妥否、当否、可否
- 恳请、拟请、望予、请予、承蒙
- 贯彻落实、组织实施、抓紧抓实、落地见效

### 严禁口头语和网络语

| 禁用 | 替换为 |
|------|--------|
| "这事儿" | "此事" |
| "咱们" | "本单位"或"我单位" |
| "撸起袖子" | "大力"或"真抓实干" |
| "干货" | "实质性内容" |
| "拍脑袋" | "随意"或"盲目" |
| "老百姓" | "群众"或"市民" |
| "一把手" | "主要负责人" |
| "没有着落" | "缺口" |

## 语言四大原则

1. **准确**：措辞恰当，概念准确，判断合乎逻辑，避免歧义
2. **平实**：语言朴实无华，不堆砌辞藻，避免溢美虚饰
3. **简明**：开门见山，删繁就简，多用短句
4. **庄重**：使用规范书面语，不用口语、方言、网络语

## 正文写作规矩

- 首段交代背景、目的或依据，再展开具体内容
- 首次出现非规范简称时，先写全称加括号注简称（如"[全称单位]（以下简称[简称]）"）
- 引用政策文件时，先写文件标题再写文号
- 二级标题下通常先用 3-5 句话概述，再用"一是、二是、三是"段内分点（不另起自然段）
- "一是、二是、三是"在同自然段内连续书写
- 多用动作导向表达："开展""落实""报送""组织实施""严格执行"
- 少用模糊词："尽快""适当""有点""比较好"→改为具体时限和标准

## 常用公文用语

**开头用语**：根据、按照、遵照、为、为了、关于

**承启用语**：现将……如下、特作如下通知、经研究决定

**结尾用语**：
- 请示：妥否，请批示 / 当否，请批示
- 报告：特此报告
- 通知：特此通知
- 函（去）：请予函复 / 函请支持为盼
- 函（复）：特此函复 / 特此函达
- 批复：特此批复
- 通报：特此通报

**期请用语**：恳请、拟请、望予、请予

**经办用语**：业经、已将、施行

## 质量检查清单（自查用）

每份公文完成后，对照以下清单自查：

### 格式自查（14 项）
- [ ] 标题是否完整（发文机关+关于+事由+的+文种）
- [ ] 结构层次序数是否按"一、（一）1.（1）"顺序
- [ ] 发文字号年份是否用六角括号〔〕
- [ ] 成文日期是否用阿拉伯数字全称
- [ ] 标点是否全角、统一
- [ ] 正文首行是否缩进 2 字符（Pt(32)，非厘米值）
- [ ] 一级标题（一、二、三）是否也设置了首行缩进 2 字符
- [ ] 主送机关是否顶格、末尾全角冒号
- [ ] 附件说明是否在正文下空一行左空 2 字
- [ ] 落款是否包含发文单位和日期
- [ ] 正文是否两端对齐（JUSTIFY）
- [ ] 西文字体是否统一为 Times New Roman
- [ ] 占位符是否替换为红色 xx
- [ ] 行距是否为固定值 28 磅

### 语言自查（8 项）
- [ ] 是否出现"首先其次最后""总而言之""众所周知"
- [ ] 是否出现口语化表达
- [ ] 是否出现网络流行语
- [ ] 是否出现强烈情感形容词
- [ ] 结尾用语是否符合文种规范
- [ ] 引用是否标注来源和依据
- [ ] 首次出现的简称是否括号注明全称
- [ ] "一是二是三是"是否在同一自然段内

### 内容自查（6 项）
- [ ] 请示是否一文一事
- [ ] 报告是否夹带请示事项
- [ ] 标题与正文内容是否一致
- [ ] 各板块分工是否清楚（背景→进展→问题→打算）
- [ ] 数据是否真实可查（或标注[待核实]）
- [ ] 附件是否完整、顺序是否正确

## 进阶：自动化深度校对

本自查清单适用于**人工快速过一遍**。如需逐字逐句的深度校对，请加载 `doc-proofread` 技能，该技能使用 DeepSeek R1 推理模型执行 7 维自动化校对：

| 维度 | doc-style（人工自查） | doc-proofread（自动校对） |
|------|----------------------|--------------------------|
| 错别字 | 快速扫读 | 逐字比对 + 同音/形近字筛查 |
| 语法 | 凭语感判断 | 句法分析 + 搭配校验 |
| 标点 | 全角/半角检查 | 逐标点规范性校验 |
| 政治表述 | 关键词检查 | 固定用语完整匹配 + 领导人信息校验 |
| 逻辑矛盾 | 难以全覆盖 | 数据交叉比对 + 时序因果关系检查 |
| AI 腔 | 凭经验识别 | 禁用词表精确匹配 + 句式模式识别 |

**建议流程**：完成写作 → 加载 `doc-style` 自查 → 加载 `doc-proofread` 深度校对 → 定稿。
