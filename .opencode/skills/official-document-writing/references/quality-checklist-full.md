# 公文质量检查清单（完整版）

> 本清单共 100+ 检查项，涵盖格式、内容、语言、逻辑、细节和发文前确认六大类。
> 公文起草完成后，请逐项核对。发现问题及时修改。

---

## 一、格式检查（40+ 项）

### 1. 用纸及版面检查

- [ ] 用纸规格：A4 型纸（210mm×297mm）
- [ ] 天头尺寸：上白边 37mm±1mm
- [ ] 订口尺寸：左白边 28mm±1mm
- [ ] 版心尺寸：156mm×225mm
- [ ] 双面印刷

### 2. 字体字号检查

- [ ] 发文机关标志：小标宋体，红色
- [ ] 文件标题：2 号小标宋体，居中排布
- [ ] 正文：3 号仿宋_GB2312
- [ ] 一级标题（一、）：3 号黑体
- [ ] 二级标题（（一））：3 号楷体_GB2312
- [ ] 三级标题（1.）：3 号仿宋_GB2312
- [ ] 四级标题（（1））：3 号仿宋_GB2312
- [ ] 发文字号：3 号仿宋_GB2312，六角括号〔〕
- [ ] 附件说明：3 号黑体
- [ ] 附注：3 号仿宋_GB2312，加圆括号
- [ ] 抄送机关：4 号仿宋_GB2312
- [ ] 印发机关和日期：4 号仿宋_GB2312
- [ ] 页码：4 号半角宋体

### 3. 结构层次检查

- [ ] 第一层："一、"（黑体）
- [ ] 第二层："（一）"（楷体_GB2312）
- [ ] 第三层："1."（仿宋_GB2312）
- [ ] 第四层："（1）"（仿宋_GB2312）
- [ ] 层次顺序严格按"一、（一）1.（1）"
- [ ] 标题回行为梯形或菱形
- [ ] 同级标题写法一致，不混用编号体系
- [ ] 各层级逐级展开，不跳级
- [ ] 10 页内控制到二级标题

### 4. 成文日期检查

- [ ] 格式：阿拉伯数字全称（如：2026 年 6 月 9 日）
- [ ] 月日不编虚位（写 1 月不写 01 月）
- [ ] 右空四字编排

### 5. 发文字号检查

- [ ] 格式：机关代字+六角括号〔〕+年份+序号+号
- [ ] 年份为 4 位阿拉伯数字全称
- [ ] 括号用六角括号〔〕，不用 [] 或【】
- [ ] 序号不加"第"字，不编虚位
- [ ] 位置：发文机关标志下空二行，居中

### 6. 格式要素完整性检查

- [ ] 份号（如需标注）
- [ ] 密级和保密期限（如需标注）
- [ ] 紧急程度（如需标注）
- [ ] 发文机关标志
- [ ] 发文字号
- [ ] 签发人（上行文）
- [ ] 标题
- [ ] 主送机关
- [ ] 正文
- [ ] 附件说明（如有）
- [ ] 发文机关署名
- [ ] 成文日期
- [ ] 印章
- [ ] 附注（如需）
- [ ] 附件（如有）
- [ ] 抄送机关（如需）
- [ ] 印发机关和印发日期
- [ ] 页码

## 二、内容检查（30+ 项，按文种）

### 1. 标题检查

- [ ] 发文机关完整准确
- [ ] 事由用"关于"引出
- [ ] 文种使用正确
- [ ] 标题简洁明了，不冗长
- [ ] 标题与正文内容一致

### 2. 主送机关检查

- [ ] 主送机关准确、使用全称或规范化简称
- [ ] 请示只有一个主送机关
- [ ] 多个主送机关按规范排序
- [ ] 位置在标题下空一行顶格
- [ ] 末尾用全角冒号

### 3. 请示类专项检查

- [ ] 一文一事
- [ ] 行文依据充分（政策+实际）
- [ ] 理由合理充分
- [ ] 请求事项明确具体
- [ ] 可行性论证充分
- [ ] 附件齐全
- [ ] 结尾用语："妥否，请批示"或"当否，请批示"
- [ ] 附注注明联系人及电话

### 4. 报告类专项检查

- [ ] 不夹带请示事项
- [ ] 基本情况交代清楚
- [ ] 数据真实准确（无则标注[待核实]）
- [ ] 问题和不足客观陈述
- [ ] 下一步打算有针对性、可落地
- [ ] 结尾用语："特此报告"

### 5. 通知类专项检查

- [ ] 制发依据充分
- [ ] 通知事项清晰具体
- [ ] 执行要求明确可操作
- [ ] 时间地点明确（会议通知）
- [ ] 责任单位明确
- [ ] 结尾用语："特此通知"

### 6. 函类专项检查

- [ ] 发函缘由说明清楚
- [ ] 商洽事项明确合理
- [ ] 语气得体（平行文）
- [ ] 答复依据充分（复函）
- [ ] 答复态度明确（复函）
- [ ] 结尾用语正确（去函"请予函复"，复函"特此函复"）

### 7. 纪要类专项检查

- [ ] 会议信息完整（时间、地点、主持人、出席、请假、列席）
- [ ] "会议认为""会议指出""会议强调""会议要求"提示语使用准确
- [ ] 议定事项明确
- [ ] 责任分工清晰（责任单位+时限）

### 8. 总结类专项检查

- [ ] 成绩总结全面（有数据支撑）
- [ ] 经验提炼有亮点
- [ ] 问题查找准确不回避
- [ ] 下一步打算可行可落地

## 三、语言检查（40+ 项）

### 1. AI 腔检测（核心项）

- [ ] 是否出现"首先……其次……最后……"
- [ ] 是否出现"总而言之""综上所述""众所周知"
- [ ] 是否出现"显而易见""毋庸置疑"
- [ ] 是否出现"从某种程度上说""在一定意义上"
- [ ] 是否出现"不仅……而且……更重要的是……"（冗余递进）

### 2. 口语化检测

- [ ] "这事儿"→改为"此事"
- [ ] "咱们"→改为"本单位"或"我单位"
- [ ] "老百姓"→改为"群众"或"市民"
- [ ] "一把手"→改为"主要负责人"
- [ ] "没有着落"→改为"存在缺口"
- [ ] "撸起袖子"→改为"真抓实干"
- [ ] "干货"→改为"实质性内容"
- [ ] "拍脑袋"→改为"随意"或"盲目"
- [ ] "打算"→改为"拟"或"决定"

### 3. 情感形容词检测

- [ ] 是否使用"英明领导"→改为"正确领导"
- [ ] 是否使用"举世瞩目"→删除或改为"显著"
- [ ] 是否使用"辉煌成就"→改为"显著成绩"
- [ ] 是否使用"排山倒海之势"→删除
- [ ] 是否使用"令人振奋""十分感人""极其重要"→删除或弱化

### 4. 简明性检测

- [ ] "进行"可删（如"进行研究"→"研究"）
- [ ] "工作"可删（如"宣传工作"→"宣传"）
- [ ] "对"可删（如"对问题进行分析"→"分析问题"）
- [ ] 多余"的"可删
- [ ] 多余"了"可删

### 5. 规范用语检测

- [ ] 依据类："根据""按照""遵照""依照"
- [ ] 目的类："为""为了"
- [ ] 承启类："现将……如下""特作如下通知"
- [ ] 期请类："恳请""拟请""望予"
- [ ] 期复类："妥否""当否""可否"
- [ ] 经办类："业经""已将"
- [ ] 谦敬类："承蒙""不胜"

### 6. 标点符号检测

- [ ] 句号为全角"。"，非英文"." 
- [ ] 逗号为全角"，"
- [ ] 无"一逗到底"现象
- [ ] 冒号为全角"："
- [ ] 发文字号六角括号为〔〕
- [ ] 引号使用正确（引用文件用书名号《》）
- [ ] 并列词语间用顿号
- [ ] 无中英文标点混用

## 四、逻辑检查（10+ 项）

- [ ] 层次分明、条理清楚
- [ ] 各层次逻辑递进
- [ ] 因果关系成立
- [ ] 总分关系合理
- [ ] 主次分明、重点突出
- [ ] 前后表述一致、无矛盾
- [ ] 文题相符
- [ ] 论据充分（有数据、事实、政策支撑）
- [ ] 推理合乎逻辑
- [ ] 各板块分工清楚（背景→进展→问题→打算）

- [ ] 职务准确
- [ ] 数字准确
- [ ] 成文日期准确
- [ ] 发文字号准确

- [ ] 附件顺序号正确
- [ ] 密级标注（如需）
- [ ] 紧急程度标注（如需）
- [ ] 公开属性确定（如需）

## 快速自查口诀

```
格式自查三件事：标题字号层次序
语言自查三件事：AI腔、口语、情感词
内容自查三件事：一文一事不夹带，标题正文要一致，板块分工要清楚
细节自查三件事：人名地名和日期，附件联系勿遗漏，标点全角勿混用
```

**使用提醒**：
1. 本清单为基本检查项，特殊公文可能有特殊要求
2. 重要公文建议多人交叉检查
3. 不确定的问题应查阅 GB/T 9704-2012 或咨询专业人员
