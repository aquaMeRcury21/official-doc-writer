# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please open an issue on GitHub.

Do not post sensitive information (API keys, personal data) in public issues.

## Scope

- API key handling in `.env` files
- Exposure of real government documents in the knowledge base
- Injection vulnerabilities in template rendering
- Network request security in `api_client.py`

## Out of Scope

- DeepSeek API platform security (report to DeepSeek directly)
- Security of third-party dependencies (report to respective projects)
